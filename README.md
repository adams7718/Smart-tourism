## 2. 專題背景與動機 (Background & Motivation)

### 2.1 專題背景
隨著自由行（FIT）成為旅遊主流，旅客不再滿足於「走馬看花」的套裝行程。然而，現有的旅遊資訊過於分散（Google Map、部落格、IG 打卡點），導致旅客在規劃行程時，往往需要耗費大量時間在不同平台間切換與比對。

### 2.2 開發動機
本小組希望利用在《系統分析與設計》課程中學到的觀念，開發一套**「智慧旅遊推薦系統」**。
主要解決以下痛點：
1.  **資訊碎片化**：解決使用者需開啟多個分頁查詢評價與路線的困擾。
2.  **缺乏個人化**：針對使用者偏好（如：喜歡文青景點、或是戶外冒險）給予權重推薦。
3.  **決策疲勞**：透過系統篩選，快速提供前 3-5 個最佳選項，輔助使用者快速決策。

---

## 3. 系統分析與架構 (System Architecture)

本系統採 B/S (Browser/Server) 架構，以前後端分離的概念進行快速原型開發。

### 3.1 系統流程圖 (Concept Flow)
1.  **Input**: 使用者輸入「旅遊地點」、「偏好類型（如美食、景點）」。
2.  **Process**: 
    * 系統呼叫 **Google Maps API (Places API)** 獲取候選名單。
    * 根據使用者偏好進行權重排序演算法 (Sorting Algorithm)。
3.  **Output**: 使用 **Streamlit** 渲染出視覺化圖表與推薦列表卡片。

### 3.2 使用技術 (Tech Stack)
* **開發語言**: Python 3.10+(豐富的第三方函式庫)
* **前端框架**: Streamlit (用於快速建構資料視覺化 Web App)
* **資料來源**: Google Maps Platform (Places API, Geocoding API)
* **資料處理**: Pandas (資料清洗與結構化)

## 4. 功能需求 (Functional Requirements)

1.  **參數設定功能**：使用者可選擇目的地城市及感興趣的景點類別。
2.  **智慧搜尋功能**：系統依據關鍵字搜尋周邊高評價地點。
3.  **資訊展示功能**：顯示景點名稱、評分、地址及照片。
4.  **地圖視覺化**：在互動式地圖上標示推薦景點的位置。
