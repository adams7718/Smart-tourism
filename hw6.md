## 圖 6：行程規劃頁(ai規劃行程)
![7](https://github.com/adams7718/Smart-tourism/blob/main/6.png)

A. 輸入欄位 (Input Fields)
| 欄位名稱 | 資料型態 | 驗證規則 / 觸發行為 |
| :--- | :--- | :--- |
| 出發城市 | 文字 (String 或 Enum) | 必填；不可空白；若使用下拉選單則限制在系統支援的城市列表 |
| 目的地城市 / 縣市 | 文字 (String 或 Enum) | 必填；需為支援的旅遊目的地；選擇後載入該地區景點資料 |
| 出發日期 | 日期 (Date) | 必填；日期不可早於今日；會影響營業時間與季節性景點篩選 |
| 旅遊天數 | 整數 (Integer) | 必填；值需介於 1–30；作為 AI 安排每日行程的基礎 |
| 每日預計旅遊時數 | 整數 (Integer, 小時) | 選填；介於 4–16 小時；未填則使用預設值 |
| 預算區間 | 整數 (Integer, 金額) | 選填；可輸入整體預算上限；影響餐飲 / 交通 / 景點選擇 |
| 旅遊主題偏好 | 列舉 (Enum，多選) | 選填；至少可選 0 個；勾選後 AI 會優先選擇符合主題的景點 |
| 交通方式偏好 | 列舉 (Enum，單選) | 選填；如「大眾運輸 / 自駕 / 步行優先」；影響路線與交通工具建議 |
| 是否避免重複景點 | 布林 (Boolean, CheckBox) | 選填；勾選時 AI 不排入重複景點 |
| 生成行程按鈕 | 按鈕 (Button) | 綜合上述條件呼叫 AI 規劃行程，產生每日景點與路線建議 |

B. 螢幕顯示欄位 (Output Fields)
| 欄位名稱 | 功能描述 |
| :--- | :--- |
| 規劃條件摘要 | 顯示使用者所選的目的地、日期、天數、主題、預算等條件 |
| AI 規劃中載入動畫 / 提示 | 顯示 AI 規劃中的狀態，避免使用者重複點擊 |
| 預估總預算 | 顯示 AI 粗略估計的整體旅遊預算 |
| 預估交通時間總計 | 顯示所有日程中交通花費時間總和 |
| 導引按鈕「查看 AI 輸出行程」 | 規劃完成後，導向下一頁顯示詳細日程列表 |

| 行程項目啟用勾選框 | 布林 (Boolean, CheckBox) | 勾選代表保留該景點；取消勾選則視為從本次行程中移除 |

| 儲存排序結果按鈕 | 按鈕 (Button) | 儲存目前排序與啟用狀態，更新行程內容 |

| 行程項目欄位：預估抵達時間 | 顯示依照排序計算後的預估抵達時間 |

| 選擇路線方案 | 列舉 (Enum) | 若 Google Map 提供多條路線，使用單選按鈕或下拉選擇其中一條，作為最終路線 |
| 開啟 Google Map 按鈕 | 按鈕 (Button) | 依目前行程順序與條件，組合 URL 並開啟新分頁顯示路線導航 |

B. 螢幕顯示欄位 (Output Fields)
| 欄位名稱 | 功能描述 |
| :--- | :--- |
| 行程路線摘要 | 顯示起點、終點與中途景點清單（出發點 → 景點 1 → 景點 2 → … → 終點） |
| 預估總距離 | 顯示整體路線預估行駛距離 |
| 預估總交通時間 | 顯示整體路線預估耗時 |
| 路線示意圖（縮圖） | 以簡略地圖或圖示顯示大致路線走向（若有嵌入 Map 預覽） |
| 導航說明文字 | 提醒使用者點擊「開啟 Google Map」後，將跳至 Google Map APP / 網頁導航 |
| 錯誤訊息（路線無法產生） | 當缺少必要經緯度或 Google Map 回傳錯誤時顯示提示，協助使用者修正條件 |
